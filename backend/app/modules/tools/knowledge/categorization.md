# Как FinAgent категоризирует траты

Каждой транзакции присваивается категория в два шага:

1. **Правила.** Название продавца из выписки (например, «PYATEROCHKA 24281 Kazan RUS») сначала
   упрощается до ключа продавца (убираются номера точек, город, страна), а затем сверяется со
   встроенным списком правил: подстрока в названии продавца сопоставляется с категорией.
   Примеры: ПЯТЁРОЧКА / MAGNIT / SPAR → «Продукты»; YANDEX.TAXI / KAZANMETRO / DELIMOBIL →
   «Транспорт»; WILDBERRIES / OZON / AVITO → «Маркетплейсы»; GOSUSLUGI → «Госуслуги».
2. **LLM для неизвестных продавцов.** Если правило не нашлось, продавец отправляется в LLM одним
   пакетным запросом вместе со списком всех доступных категорий — LLM выбирает наиболее подходящую
   или ставит «Прочее», если не уверена.

Результат классификации (правило или ответ LLM) **кэшируется** в таблице продавцов, поэтому один и
тот же продавец никогда не отправляется в LLM повторно — при следующей встрече категория берётся
из кэша.

Пользователь может вручную переназначить категорию продавцу в настройках — это обновит кэш и
изменит категорию для всех будущих транзакций этого продавца.
